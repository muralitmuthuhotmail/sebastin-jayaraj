Customize Findoora Docs to match your brand, requirements, and workflow.

## Site Configuration

The main site configuration is located in `lib/config.ts`. Update these values to customize your documentation site:

```typescript
// lib/config.ts
export const SITE_NAME = "Your Docs Site";
export const SITE_SUB_TITLE = "Documentation";
export const SITE_DESCRIPTION = "Your comprehensive documentation hub";
export const GITHUB_URL = "https://github.com/your-username/your-repo";
export const SITE_AUTHOR = "@your-handle";
export const SITE_KEYWORDS = ["documentation", "guides", "tutorials"];

// Navigation
export const ENABLE_GITHUB_LINK = true;
export const ENABLE_THEME_TOGGLE = true;
export const ENABLE_SEARCH = true;

// Content
export const POSTS_PER_PAGE = 10;
export const ENABLE_COMMENTS = false;
export const ENABLE_SOCIAL_SHARE = true;

// SEO
export const ENABLE_ANALYTICS = true;
export const ENABLE_SEO_OPTIMIZATION = true;
```

## Environment Variables

### Development Environment

`.env.local`

```env
# Base URLs
NEXT_PUBLIC_BASE_URL=http://localhost:3000
NEXT_PUBLIC_BASE_API_URL=http://localhost:3000

# Analytics (Optional)
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX

# Google Services (Optional)
GOOGLE_SITE_VERIFICATION=your-verification-code

# Performance
NEXT_TELEMETRY_DISABLED=1

# Development
NODE_ENV=development
```

### Production Environment

```env
# Base URLs (Update with your domain)
NEXT_PUBLIC_BASE_URL=https://your-domain.com
NEXT_PUBLIC_BASE_API_URL=https://your-domain.com

# Analytics
NEXT_PUBLIC_GA_ID=your-google-analytics-id

# SEO
GOOGLE_SITE_VERIFICATION=your-verification-code

# Performance
NODE_ENV=production
NEXT_TELEMETRY_DISABLED=1

# Security (Optional)
NEXTAUTH_SECRET=your-secret-key
NEXTAUTH_URL=https://your-domain.com
```

### Docker Environment Variables

In your `docker-compose.yml`:

```yaml
version: "3.8"
services:
  docs:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - NEXT_PUBLIC_BASE_URL=https://your-domain.com
      - NEXT_PUBLIC_GA_ID=your-ga-id
      - NEXT_TELEMETRY_DISABLED=1
    restart: unless-stopped
```

```css
  --secondary: 210 40% 96%;

  --muted-foreground: 215.4 16.3% 46.9%;

  --accent-foreground: 222.2 47.4% 11.2%;

  --input: 214.3 31.8% 91.4%;
```

```typescript
export const SEARCH_CONFIG = {
  // Search behavior
  enableLiveSearch: true,
  enableSearchSuggestions: true,
  enableCategoryFiltering: true,
  enableTagFiltering: true,
```

## Security Configuration

Configure CSP in `next.config.ts`:

```typescript
const securityHeaders = [
  {
    key: "Content-Security-Policy",
    value: `
      default-src 'self';
      script-src 'self' 'unsafe-eval' 'unsafe-inline' *.googletagmanager.com;
      style-src 'self' 'unsafe-inline';
      img-src 'self' data: blob: *.unsplash.com;
      font-src 'self';
      object-src 'none';
      base-uri 'self';
      form-action 'self';
      frame-ancestors 'none';
      upgrade-insecure-requests;
    `
      .replace(/\s{2,}/g, " ")
      .trim(),
  },
];
```

## Backup & Restore

### Configuration Backup

Create a backup script for your configuration:

```bash
#!/bin/bash
# backup-config.sh

# Create backup directory
mkdir -p backups/$(date +%Y%m%d)

# Backup configuration files
cp lib/config.ts backups/$(date +%Y%m%d)/
cp .env.local backups/$(date +%Y%m%d)/
cp next.config.ts backups/$(date +%Y%m%d)/
cp tailwind.config.js backups/$(date +%Y%m%d)/
cp package.json backups/$(date +%Y%m%d)/

echo "Configuration backed up to backups/$(date +%Y%m%d)/"
```

## Configuration Validation

### Validation Script

Create a script to validate your configuration:

```typescript
// scripts/validate-config.ts
import { SITE_NAME, GITHUB_URL } from "@/lib/config";

function validateConfig() {
  const errors = [];

  if (!SITE_NAME || SITE_NAME === "Your Docs Site") {
    errors.push("SITE_NAME not configured");
  }

  if (!process.env.NEXT_PUBLIC_BASE_URL) {
    errors.push("NEXT_PUBLIC_BASE_URL not set");
  }

  if (errors.length > 0) {
    console.error("Configuration errors:", errors);
    process.exit(1);
  }

  console.log("✅ Configuration is valid");
}

validateConfig();
```

## Configuration Help

Common configuration issues:

- **Environment variables not loading**: Check file naming (`.env.local` for development)
- **Styles not applying**: Verify Tailwind configuration and CSS imports
- **Features not working**: Check feature toggles in `lib/config.ts`
- **Build failures**: Run `pnpm typecheck` to identify TypeScript issues

Need more help? Check the [Troubleshooting Guide](./troubleshooting) or [open an issue](https://github.com/muralitmuthuhotmail/findoora-docs/issues).
