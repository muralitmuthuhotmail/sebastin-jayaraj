Common issues and their solutions for Findoora Docs.

## Installation Issues

### Node.js Version Problems

**Problem**: Build fails with Node.js version errors

```bash
Error: The engine "node" is incompatible with this module.
```

**Solution**:

```bash
# Check current Node.js version
node --version

# Install Node.js 18+ using nvm
nvm install 18
nvm use 18

# Or update using package manager
# macOS with Homebrew
brew install node@18

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### pnpm Installation Issues

**Problem**: `pnpm` command not found

```bash
bash: pnpm: command not found
```

**Solution**:

```bash
# Install pnpm globally
npm install -g pnpm

# Or use alternative installation methods
curl -fsSL https://get.pnpm.io/install.sh | sh -

# Verify installation
pnpm --version
```

**Problem**: Permission denied when installing pnpm globally

```bash
Error: EACCES: permission denied
```

**Solution**:

```bash
# Option 1: Use npm prefix
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH

# Option 2: Use node version manager
nvm install node  # Installs latest Node.js
npm install -g pnpm

# Option 3: Use yarn instead
npm install -g yarn
yarn install  # Instead of pnpm install
```

### Dependency Resolution Issues

**Problem**: Package dependency conflicts

```bash
Error: Conflicting peer dependencies
```

**Solution**:

```bash
# Clear package manager cache
pnpm store prune
rm -rf node_modules
rm pnpm-lock.yaml

# Reinstall dependencies
pnpm install

# If still failing, try legacy peer deps
pnpm install --legacy-peer-deps

# Or force resolution
pnpm install --force
```

## Build Issues

### TypeScript Compilation Errors

**Problem**: TypeScript build failures

```bash
Type error: Property 'xyz' does not exist on type
```

**Solution**:

```bash
# Run type checking to see detailed errors
pnpm typecheck

# Clear TypeScript cache
rm -rf .next/cache
rm -rf node_modules/.cache

# Check for missing type definitions
pnpm add -D @types/node @types/react @types/react-dom

# Restart TypeScript server in VS Code
# Command Palette > TypeScript: Restart TS Server
```

**Common TypeScript fixes**:

```typescript
// Fix: Add proper typing for custom props
interface CustomComponentProps {
  children: React.ReactNode;
  className?: string;
}

// Fix: Use proper Next.js types
import type { Metadata } from 'next';
import type { NextRequest, NextResponse } from 'next/server';

// Fix: Type MDX components properly
const components: MDXComponents = {
  h1: (props) => <h1 className="text-4xl font-bold" {...props} />,
  // ...
};
```

### MDX Processing Errors

**Problem**: MDX files not rendering correctly

```bash
Error: Could not parse expression with acorn
```

**Solution**:

```bash
# Check MDX file syntax
# Ensure all JSX is properly closed
<Component>
  Content here
</Component>

# Not: <Component> Content here (missing closing tag)

# Check for conflicting frontmatter
---
title: "My Title"  # Use quotes for titles with special characters
date: "2025-07-26"  # Use consistent date format
---

# Verify MDX component imports
import { CustomComponent } from '../components/custom';

<CustomComponent prop="value">
  Content
</CustomComponent>
```

### Build Memory Issues

**Problem**: Build fails with out of memory errors

```bash
FATAL ERROR: Ineffective mark-compacts near heap limit
```

**Solution**:

```bash
# Increase Node.js memory limit
NODE_OPTIONS="--max_old_space_size=4096" pnpm build

# Or set in package.json
{
  "scripts": {
    "build": "NODE_OPTIONS='--max_old_space_size=4096' next build"
  }
}

# For very large sites, consider incremental builds
# Enable experimental features in next.config.ts
experimental: {
  incrementalCacheHandlerPath: require.resolve('./cache-handler.js'),
}
```

## Docker Issues

### Container Won't Start

**Problem**: Docker container exits immediately

```bash
docker: Error response from daemon: container exited with code 1
```

**Solution**:

```bash
# Check container logs
docker logs container-name

# Common fixes:
# 1. Check Dockerfile syntax
# 2. Verify port exposure
EXPOSE 3000

# 3. Check entry point
CMD ["pnpm", "start"]
# or
CMD ["node", "server.js"]

# 4. Verify file permissions
RUN chown -R nextjs:nodejs /app
USER nextjs
```

### Port Already in Use

**Problem**: Port 3000 is already in use

```bash
Error: listen EADDRINUSE: address already in use :::3000
```

**Solution**:

```bash
# Find what's using the port
lsof -i :3000

# Kill the process (replace PID)
kill -9 <PID>

# Or use a different port
docker run -p 3001:3000 findoora-docs

# For docker-compose, update ports
services:
  docs:
    ports:
      - "3001:3000"
```

### Docker Build Failures

**Problem**: Docker build fails with network errors

```bash
ERROR: failed to solve: failed to load cache key
```

**Solution**:

```bash
# Clear Docker cache
docker system prune -a

# Build without cache
docker build --no-cache -t findoora-docs .

# Check Docker BuildKit version
docker buildx version

# Use legacy builder if needed
DOCKER_BUILDKIT=0 docker build -t findoora-docs .
```

```bash
  "outputDirectory": ".next"

# 4. Check dependencies
# Ensure all dependencies are in package.json, not just devDependencies
```

### Static Export Issues

**Problem**: Static export fails with dynamic routes

```bash
Error: Dynamic server usage not allowed in static export
```

**Solution**:

```typescript
// Ensure all dynamic routes have generateStaticParams
export async function generateStaticParams() {
  const posts = await getContentPosts();
  return posts.map((post) => ({
    category: post.category,
    slug: post.slug,
  }));
}

// Remove dynamic server functions in static export
// Comment out or remove:
// - headers()
// - cookies()
// - searchParams in Server Components
```

## Content Issues

### Content Not Showing

**Problem**: MDX content not appearing on site

**Solution**:

```bash
# Check file location
# Ensure files are in correct directory:
posts/my-post.mdx
# or
md-content/my-post.mdx

# Check frontmatter syntax
---
title: "My Title"        # Required

# Check category spelling
```

**Problem**: CSS custom properties not applying

**Solution**:

```css

:root {
  --primary: 200 100% 50%; /* HSL values without hsl() */

.text-primary {
  color: hsl(var(--primary));
}

/* Check dark mode configuration */
[data-theme="dark"] {
  --primary: 200 100% 70%; /* Lighter for dark mode */
}
```

## Performance Issues

### Slow Build Times

**Problem**: Build takes too long

**Solution**:

```bash
# Enable Turbopack (experimental)
pnpm dev --turbo

# Analyze bundle size
pnpm build:analyze

# Remove unused dependencies
pnpm dlx depcheck

# Optimize images
# Use next/image component
# Enable image optimization in next.config.ts

# Enable SWC minifier
// next.config.ts
swcMinify: true

# Consider incremental builds
experimental: {
  incrementalCacheHandlerPath: './cache-handler.js'
}
```

### Large Bundle Size

**Problem**: JavaScript bundle too large

**Solution**:

```bash
# Analyze bundle
pnpm build:analyze

# Common optimizations:
# 1. Dynamic imports for heavy components
const HeavyComponent = dynamic(() => import('./heavy-component'), {
  loading: () => <Skeleton />,
});

# 2. Remove unused dependencies
# Check bundle analyzer for largest packages

# 3. Tree shaking
# Use ES modules imports
import { specific } from 'library';
# Not: import * as library from 'library';

# 4. Code splitting at route level (automatic with App Router)
```

### Memory Leaks

**Problem**: High memory usage during development

**Solution**:

```bash
# Restart development server regularly
pnpm dev

# Check for:
# 1. Circular dependencies
# 2. Event listeners not being cleaned up
useEffect(() => {
  const handler = () => {};
  window.addEventListener('event', handler);

  return () => {
    window.removeEventListener('event', handler);  # Cleanup
  };
}, []);

# 3. Large state objects
# Use pagination for large lists
# Implement virtual scrolling for many items
```
